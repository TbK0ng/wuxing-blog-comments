## 📋 基础插件配置模板

### 1. 最小化插件模板
```json
{
    "name": "my-minimal-plugin",
    "displayName": "My Minimal Plugin",
    "description": "A minimal ClaudeCode plugin",
    "version": "1.0.0",
    "author": "Your Name",
    "components": {
        "commands": []
    }
}
```

## 📝 配置说明

### 配置字段详解

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `name` | string | ✅ | 插件唯一标识，使用kebab-case |
| `displayName` | string | ✅ | 插件显示名称 |
| `description` | string | ✅ | 插件功能描述 |
| `version` | string | ✅ | 遵循语义化版本（semver） |
| `author` | string | ✅ | 作者或团队名称 |
| `license` | string | ❌ | 开源许可证类型 |
| `repository` | string | ❌ | GitHub仓库URL |
| `components` | object | ✅ | 插件组件定义 |
| `dependencies` | object | ❌ | 依赖项声明 |
| `configuration` | object | ❌ | 默认配置设置 |

### components字段说明

```json
{
    "components": {
        "commands": ["cmd1", "cmd2"],        // 命令文件名列表
        "agents": ["agent1", "agent2"],      // Agent文件名列表
        "mcpServers": ["server1", "server2"], // MCP服务器名列表
        "hooks": {                           // Hook配置映射
            "event-name": "hook-script-name"
        }
    }
}
```

### 最佳实践

1. **命名规范**
   - 插件名使用kebab-case：`my-awesome-plugin`
   - 版本遵循semver：`1.0.0`, `1.1.0`, `2.0.0`
   - 显示名称使用Title Case：`My Awesome Plugin`

2. **版本管理**
   - 主版本号：不兼容的API修改
   - 次版本号：向下兼容的功能性新增
   - 修订号：向下兼容的问题修正

3. **依赖声明**
   - 明确列出所有必需的工具
   - 使用版本范围避免冲突
   - 提供可选依赖的说明

4. **配置设置**
   - 提供合理的默认值
   - 使用环境变量存储敏感信息
   - 记录所有配置选项的用途

## 🚀 快速开始
