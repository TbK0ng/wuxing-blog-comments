# ClaudeCode核心命令速查表

## 📋 基础管理命令

| 命令 | 语法 | 功能 | 使用场景 | 示例 |
|------|------|------|----------|------|
| `/help` | `/help` | 显示帮助信息 | 初期学习、命令查询 | `/help` |
| `/plan` | `/plan` | 进入计划模式 | 复杂任务分解规划 | `/plan` |
| `/model` | `/model <model-name>` | 切换AI模型 | 根据任务复杂度选择模型 | `/model claude-sonnet-4-5-20250929` |
| `/context` | `/context` | 查看上下文使用 | 监控资源使用 | `/context` |
| `/add-dir` | `/add-dir <directory>` | 添加目录到上下文 | 精确控制AI理解范围 | `/add-dir src/components` |
| `/compact` | `/compact` | 压缩对话历史 | 优化上下文大小 | `/compact` |
| `/clear` | `/clear` | 清除对话历史 | 重新开始新话题 | `/clear` |

## ⚠️ 关于插件系统的说明

**重要提醒：** ClaudeCode的插件生态系统目前还在建设初期，可用插件有限。
建议重点关注核心命令的熟练使用，而非依赖插件扩展功能。

如果确实需要插件功能，可以通过以下方式：
```bash
/plugin marketplace        # 查看当前可用插件
/plugin install <name>     # 安装指定插件
```

## 💾 Git工作流集成

ClaudeCode没有专门的Git斜杠命令，而是通过自然语言与Git集成：

| 操作 | ClaudeCode命令示例 | 功能 | 使用场景 |
|------|-------------------|------|----------|
| 查看状态 | `claude "check git status"` | 查看工作区状态 | 了解当前变更情况 |
| 提交代码 | `claude "commit changes with message: feat: add user authentication"` | Git提交 | 版本控制 |
| 查看差异 | `claude "show git diff for staged changes"` | 查看代码变更 | 代码审查 |
| 创建分支 | `claude "create new branch feature/user-login and switch to it"` | 分支管理 | 功能开发 |
| 合并分支 | `claude "merge feature branch into main"` | 分支合并 | 完成功能开发 |

## 📝 使用技巧

### 命令补全
- 使用Tab键快速补全命令和文件路径
- 输入部分命令后按Tab查看可用选项

### 历史搜索
- Ctrl+R：搜索历史命令
- ↑/↓：浏览命令历史

### 上下文管理
- 定期使用`/compact`清理对话历史
- 使用`@-mention`精确指定文件范围
- 使用`/context`检查当前上下文使用情况

### MCP服务器
- 预先配置需要的MCP服务器
- 使用`@-mention`调用外部服务
- 定期检查MCP服务器状态

## 🚨 注意事项

### 安全提醒
- 在生产环境中谨慎使用自动化命令
- 定期检查和更新API密钥权限
- 使用沙盒模式处理不确定的代码

### 性能优化
- 合理管理上下文大小，避免包含不必要的文件
- 定期使用`/compact`命令清理对话历史
- 根据任务复杂度选择合适的模型

### 最佳实践
- 建立个人命令使用习惯
- 创建项目特定的配置模板
- 定期更新技能和知识
